Flow 1 · One-time setup

Setting up Nugen on the platform

Do this once, when you start. About 90 minutes for a 13-person team. After this, you're set — payroll "just works" every month.

1

Sign in & create the organisation

Owner enters email → gets a 6-digit code → fills in the org wizard.

▼
📍 Where
Open the app → click Sign in → enter cto.varun@gmail.com → check email → enter the 6-digit code.
⌨️ What to type
Org name: Nugen IT Services · Country: India · Timezone: Asia/Kolkata · Size: 11–50.
✅ What should happen
You land on the dashboard with a "Welcome, Varun!" banner. Sidebar shows Directory, Attendance, Leave, Payroll, Settings.
⚠️ Common mistake
If the OTP email isn't arriving, check the dev mail catcher at localhost:8025. Don't repeatedly click "Resend" — each click invalidates the previous code.
2

Fill in business details (PAN, GSTIN, registered address)

Settings → Business. These show up on every payslip and Form 16.

▼
📍 Where
Sidebar → Settings → Business
⌨️ What to fill in
Legal name, CIN, PAN (10-char), GSTIN, registered address (with PIN), TAN (for TDS), signing authority's full name + designation (used on Form 24Q).
⚠️ Don't skip this
Without PAN + registered address, statutory reports (PF ECR, Form 16) won't validate when you try to file them.
✅ Result
Your company name + address now appears in the header of every payslip you generate.
3

Set working days, hours, and holidays

Settings → Work Preferences. Drives attendance, LOP, overtime calculations.

▼
📍 Where
Settings → Work Preferences
⌨️ Recommended for Nugen
Working days: Mon–Fri · Hours: 09:30–18:30 · Public holidays for FY 2026–27: 26 Jan, 15 Aug, 2 Oct, Diwali, plus 4 floating.
💡 Why it matters
A "21 working days" month and a "23 working days" month produce different per-day rates. This setting tells the calc engine which days count.
⚠️ Watch out
If you change working days mid-financial-year, only future runs are affected. Past payslips are immutable — they keep their original calculation.
4

Configure statutory rates (PF, ESI, PT, LWF, TDS)

Settings → Payroll. The most important screen. Six sub-tabs.

▼
📍 Where
Settings → Payroll
✅ Pre-filled defaults for India
PF: 12% employee + 12% employer, ₹15,000 wage ceiling
ESI: 0.75% / 3.25%, ₹21,000 ceiling
PT (Punjab): ₹200 flat
TDS: off until investment declarations come in
🧮 What this drives
Every monthly run reads these rates. PF is capped at the ₹15k ceiling — so even an employee on ₹50k basic only contributes ₹1,800 (= 15,000 × 12%).
⚠️ One subtle thing
Whether you type 12 or 0.12 for the PF rate, the system is smart enough to handle both. (The internal engine uses fractions; the UI accepts percentages.)
5

Turn the Payroll feature on

Settings → Features → toggle "Payroll" to ON.

▼
📍 Where
Settings → Features → flip the toggle next to "Payroll"
✅ Result
The "Payroll" item appears in your sidebar. Without this flag, clicking Payroll shows a "Feature locked — contact admin" screen.
6

Add your 13 employees

Directory → Add Employee. Capture identity, employment, statutory, bank.

▼
📍 Where
Sidebar → Directory → click + Add Employee (or import a CSV with the "Bulk Import" button)
📋 Required fields per employee
Identity: name, work email, phone, DOB
Employment: dept, designation, joining date, reporting manager
Statutory: PAN, UAN, PF account #
Bank: account #, IFSC, account holder name
⚠️ Joining date matters
Someone joining on the 14th of a month gets paid only for days 14–end. The platform auto-prorates. So get the joining date right!
💡 Tip
Bulk-import from a CSV. We provide a template at "Directory → Import → Download template". Fastest way to add 13 people in 5 minutes.
7

Build a salary structure for each employee

Payroll → Salary → +New. Use the 50/30/20 split for tax efficiency.

▼
📍 Where
Payroll → Salary → + New Structure
🧮 The 50/30/20 split
For an employee on ₹10.26 LPA:
BASIC = 50% = ₹5,13,000/yr (₹42,750/mo)
HRA = 30% = ₹3,07,800/yr (₹25,650/mo)
SPECIAL = 20% = ₹2,05,200/yr (₹17,100/mo)
💡 Why this split
HRA gets partial tax exemption. Basic drives PF (capped at ₹15k anyway). Special is the "balancing" component. This is the safest, most tax-efficient default in India.
⚠️ Status: draft → active
A new structure starts in draft. You must click Submit for Approval → then someone (the owner) clicks Approve. Only active structures are picked up by payroll.
8

Set up leave types and policies

Leaves → Policies. Define Casual, Sick, Earned, WFH, Comp-off rules.

▼
📍 Where
Sidebar → Leaves → Policies tab (admin only)
📋 Recommended leave types for Nugen
Casual (CL): 12/yr, no carry-forward
Sick (SL): 8/yr, no carry-forward
Earned (EL): 15/yr, accrue 1.25/mo, max 30 carry-forward, encashable
WFH: unlimited, manager pre-approval, doesn't deduct from any bucket
Comp-off: earned by working weekends, 90-day expiry
✅ Done — you're set
That's the entire one-time setup. From here on it's daily HR (next flow) and monthly payroll (the flow after).

Flow 3 · Once a month

A payroll run, from "Initiate" to "Salary credited"

This is the same workflow we just executed for Nugen's March 2026 cycle. 8 steps, ~30 minutes of HR time once you're set up. Notice the status changes at every step.

draft → review → approved → finalized → paid
1

Initiate the run draft

Last working day. Pick the month, click "Initiate". 30 seconds.

▼
📍 Where
Payroll → click + New Run → pick March 2026 → Submit
✅ What happens
System finds all 13 active employees, creates a payroll record for each (no numbers yet), sets run status to draft. You see "PR-2026-03-001" in the list.
2

Process the run review

Click "Process". Calc engine fires. ~5 seconds for 13 employees.

▼
📍 Where
Open the run → click Process button
⚙️ What the engine does (per employee)
1. Reads salary structure → builds earnings list
2. Reads attendance → applies LOP and joining-date proration
3. Computes statutory: PF (capped ₹1,800), ESI (skipped if > ₹21k), PT (₹200), TDS
4. Sums up: Net = Gross − Deductions + Reimbursements + Bonuses + OT
✅ Result
Run flips to review. Each employee row now shows full breakdown — earnings, deductions, statutory, totals.
⚠️ If an employee is missing
Their salary structure is still in draft. Go to Payroll → Salary → find them → Submit + Approve → re-process the run.
3

Review entries review

Eyeball the grid. Check for surprises. Override or hold if needed.

▼
👁️ What to check
Total gross looks reasonable (~₹12.87L for Nugen)? · No negative net pay anywhere? · PF is ₹1,800 for everyone with basic ≥ ₹15k? · New joiners' gross is prorated as expected?
🛠️ Per-employee actions
Override: edit a single component (e.g. add ₹10k bonus). Logged in audit trail.
Hold: skip this employee from finalize (e.g. dispute pending).
Release: undo a hold.
⚠️ Common surprise
Gross is unexpectedly low → check the joining date. Mid-month joiner gets prorated, which is correct but easy to forget.
4

Approve approved

Owner clicks "Approve". Locks the numbers for finalize.

▼
👤 Who can do this
Org owner or admin. The "maker" (HR who processed) and the "checker" (approver) are ideally different people. For Nugen with one admin, owner-bypass is allowed and logged.
📍 Where
Open the run → Approve button → confirm modal
5

Finalize finalized

Locks everything. After this, you can't edit numbers — only mark paid.

▼
📍 Where
Open the run → Finalize button
⚠️ Point of no return
Once finalized, payroll numbers are immutable. To fix a mistake, you need to "Reopen" the run (admin-only, audited) which resets it back to review.
✅ What happens
Year-to-date counters update. Statutory accruals get posted. Form 16 data sources lock for the month.
6

Generate payslips

One-click button. Creates 13 payslip records. PDF renders on download.

▼
📍 Where
Open the run → Generate Payslips
✅ Result
13 payslip records created. Each one is a snapshot — even if you edit the employee's name tomorrow, the payslip preserves the name as it was when generated.
📄 The PDF itself
1-page A4. Header (org name + address). Employee block (name, ID, PAN, UAN, designation). Earnings table. Deductions table. Net payable in figures + words. YTD summary. Footer.
7

Distribute payslips + bank file

Email blast OR self-service. Download bank file → upload to your bank's payroll portal.

▼
📤 Distribution options
A. Email blast — one button, sends per-person download link.
B. Self-service — employees go to "Payroll → Payslips" and download themselves.
Most teams use both.
🏦 Bank file
Open run → Bank File → "Validate" first (catches missing IFSCs) → "Download CSV" → upload to your bank's payroll portal (HDFC ENet, ICICI CIB, etc.).
⚠️ Validate before download
The validate step shows you which employees have missing or malformed bank details. Fix them in Directory before downloading the CSV — banks reject the whole file otherwise.
8

Mark Paid paid

After bank confirms transfer. Triggers "Salary credited" notifications.

▼
📍 Where
Open the run → Mark Paid → enter UTR / transaction reference
✅ Result
Run status flips to paid. Every employee gets a "Your March salary has been credited" notification (email + in-app). The cycle is complete.
Flow 4 · Compliance

Statutory reports — when each one fires

India compliance has four standard filings. The platform generates the right file format for each. You download and upload it to the relevant government portal.

Report Frequency Deadline What it covers Where to file
PF ECR Monthly 15th of next month Per-employee PF contribution + UAN EPFO unified portal
ESI Return Monthly 15th of next month Per-employee ESI (only employees ≤ ₹21k) ESIC portal
Form 24Q Quarterly Jul 31 / Oct 31 / Jan 31 / May 31 YTD TDS deducted, employee-wise breakdown TIN-NSDL
Form 16 Annual 15 June FY-wide salary + tax cert per employee Email each employee
For Nugen, March 2026: PF ECR will show ₹1,800 × 13 employees + ₹1,800 × 13 employer + ₹75 × 13 admin + ₹75 × 13 EDLI = ~₹49,500 to remit by April 15. ESI: nil (everyone above ₹21k ceiling). 24Q-Q4: nil for the demo (no TDS yet). Form 16 generates June 2027.

📖 Reference

Glossary — every acronym, in plain English

Indian payroll has a lot of three-letter abbreviations. Here they all are, with a real example for each.

Provident Fund PF / EPF

Mandatory retirement savings. 12% from employee + 12% from employer go into a government fund (EPFO). Employee gets it back at retirement (or on switching jobs).

All Nugen employees with basic ≥ ₹15k contribute ₹1,800/month — the maximum capped amount.

Employee State Insurance ESI

Health insurance for low-income workers. Applies only if gross ≤ ₹21k/month. Employee pays 0.75%, employer pays 3.25%.

No Nugen employee qualifies — everyone earns above ₹21k. ESI is ₹0 across the board.

Professional Tax PT

A small state-level tax. Slabs vary by state. Punjab: flat ₹200/month for employees earning ≥ ₹15k.

Every Nugen payslip shows PT = ₹200, going to Punjab state.

Tax Deducted at Source TDS

Income tax that the employer deducts from each paycheck on the employee's behalf. Calculated based on annual income + declared investments.

Currently ₹0 for Nugen — until employees submit FY26-27 declarations in April.

Cost To Company CTC

The total annual cost of an employee to the company — including base salary, allowances, employer's PF/ESI contributions, gratuity, bonuses.

Abhishek's CTC ≈ ₹10.5L. His take-home is lower (~₹83.5k × 12 = ₹10.02L) because of PF + PT deductions.

Gross Salary

Total earnings (Basic + HRA + Special + any bonuses + reimbursements) before any deductions.

Varun's gross = ₹2,85,000/month. This is the top-line number on his payslip.

Net Pay / Take-Home

What actually hits the employee's bank account. Gross − Deductions. The number that matters most to employees.

Varun: ₹2,85,000 gross − ₹2,000 deductions = ₹2,83,000 net.

House Rent Allowance HRA

A salary component meant to cover rent. Tax-exempt up to a calculated limit (depends on rent paid + city).

For an employee on ₹10.26L CTC with 30% HRA, that's ₹25,650/month. Some of it is tax-free.

Special Allowance

A flexible "balancing" salary component. Fully taxable. Used to make the basic + HRA add up to the agreed CTC.

Special is the leftover after Basic (50%) and HRA (30%) — usually ~20% of CTC.

Loss of Pay LOP

Days an employee was absent without paid leave. The platform deducts: gross × LOP days ÷ total working days.

If a 22-working-day month has 2 LOP days, gross is reduced by 2/22 ≈ 9%.

Universal Account Number UAN

A 12-digit number that uniquely identifies each PF account holder across job switches. Issued by EPFO.

Required field on every employee record. Without it, PF ECR file fails validation.

Permanent Account Number PAN

10-character alphanumeric tax ID issued by the Income Tax Department. Format: 5 letters + 4 digits + 1 letter.

Required for TDS. Mandatory on Form 16. Without it, employees can't claim refunds.

PF Wage Ceiling

The maximum salary base on which PF contribution is calculated — currently ₹15,000/month.

An employee earning ₹50k basic still only contributes 12% × ₹15,000 = ₹1,800. The ₹35k difference is ignored.

Maker–Checker

A control where one person does an action and a different person approves it. Reduces fraud and mistakes.

HR processes a payroll run (maker), Owner approves it (checker). Single-admin orgs can owner-bypass.

Form 16

An annual TDS certificate the employer gives every employee for each financial year. Used for filing personal income tax returns.

Generated by 15 June every year. Each Nugen employee gets one PDF for FY 2026–27 in June 2027.

Joining-Date Proration

If an employee joins mid-month, they get paid only for days from joining to month-end.

Joins on March 14 (a 31-day month) → gets paid 18/31 ≈ 58% of full gross for March.

Troubleshooting — the 8 problems you'll actually hit

From running payroll for Nugen across testing + the live demo, these are the snags HR teams most commonly run into. Each has a one-line fix.

!

Payroll run says "0 employees processed"

Most common issue on first run. Fix in 30 seconds.

▼
🔍 Why
All salary structures are stuck in draft. The engine only picks up active structures.
✅ Fix
Payroll → Salary → for each employee: Submit for Approval → Approve. Then re-process the run.
!

An employee's gross is way lower than expected

Check joining date and LOP days first.

▼
🔍 Why
Either (a) joining date is mid-month → proration kicks in, or (b) attendance has unapproved LOP days.
✅ Fix
Open the entry → look at "Days breakdown". If LOP > 0 was accidental, approve the leave/regularisation in HR module → re-process the run.
!

PF deduction is wildly wrong (₹0 or ₹50k+)

Almost always a config issue.

▼
🔍 Why
Settings → Payroll → PF tab — the rate or wage-ceiling is misconfigured, OR the salary structure has no component flagged "isStatutory".
✅ Fix
Reset PF config to defaults: 12% / 12% / ₹15,000 ceiling. Make sure BASIC component on each structure has isStatutory: true.
!

Bank file rejected — invalid IFSC

One bad row → bank rejects entire batch.

▼
🔍 Why
An employee's IFSC has wrong characters or length, or account number has spaces.
✅ Fix
Always click "Validate Bank File" before downloading. The validator lists which employees have malformed details. Fix in Directory, re-download.
!

Need to fix a number after Finalize

There's a "Reopen" button — admin only, audited.

▼
🔍 Why
Finalize is meant to be the point of no return — but mistakes happen.
✅ Fix
Open the run → "Reopen" → confirm with reason. Status reverts to review. Make the fix → re-Approve → re-Finalize. Audit trail logs the reopen reason.
!

Generate Payslips fails with "duplicate key"

Someone already generated them — they exist.

▼
🔍 Why
A previous click already created the payslip records. The system enforces "one payslip per employee per month" via a database index.
✅ Fix
No fix needed — the payslips ARE generated. Go to Payroll → Payslips list and download / share them.
!

Manager can't see leave requests from a team member

Reporting-manager link missing on the employee record.

▼
🔍 Why
Leave routing uses the "Reporting Manager" field on the employee's Directory record. If empty, leave requests go to HR/admin queue instead.
✅ Fix
Directory → open the employee → set "Reporting Manager" → save. New requests now route correctly. Past requests stay where they were.
!

"Net payable in words" looks wrong on payslip

Known issue with the words formatter — being fixed.

▼
🔍 Why
The "amount in words" line drops thousands. ₹45,500 prints as "Rupees Four Hundred and Fifty Five Only".
⏳ Status
Already flagged for fix. The numerical net pay is correct — only the text rendering is wrong.
Stuck on something not listed? Take a screenshot of what you see and what you expected, and ping the platform team. Most issues are config drift — fixing them takes minutes once we see the screen.
